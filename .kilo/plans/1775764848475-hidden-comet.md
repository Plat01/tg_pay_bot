# План исправления ошибки PaymentStatusError

## Проблема

При проверке статуса платежа возникает ошибка:
```
name 'PaymentStatusError' is not defined
```

Ошибка возникает в файле `src/infrastructure/payments/platega.py:336`, где используется `PaymentStatusError`, но этот класс не импортирован.

## Решение

Добавить `PaymentStatusError` в импорты в файле `src/infrastructure/payments/platega.py`.

### Изменения

**Файл: `src/infrastructure/payments/platega.py`**

Строки 30-35 (текущий импорт):
```python
from src.infrastructure.payments.exceptions import (
    PaymentCreationError,
    PaymentProviderUnavailable,
    PaymentSignatureError,
    PaymentValidationError,
)
```

Заменить на:
```python
from src.infrastructure.payments.exceptions import (
    PaymentCreationError,
    PaymentProviderUnavailable,
    PaymentSignatureError,
    PaymentStatusError,
    PaymentValidationError,
)
```

## Проверка

1. Класс `PaymentStatusError` определен в `src/infrastructure/payments/exceptions.py:16`
2. Класс экспортируется в `src/infrastructure/payments/__init__.py:20`
3. После добавления импорта ошибка исчезнет

## Влияние

- Минимальное изменение (добавление одной строки в импорт)
- Не влияет на другие части кода
- Исправляет критическую ошибку, блокирующую проверку статуса платежа
