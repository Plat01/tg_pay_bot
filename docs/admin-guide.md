# Инструкция по использованию Telegram Pay Bot (для администратора)

## Введение

**Telegram Pay Bot** — это асинхронный Telegram бот для продажи VPN подписок с интегрированной платежной системой и реферальной программой.

### Основные возможности

- Продажа VPN подписок (4 тарифа: trial, monthly, quarterly, yearly)
- VPN подписки генерируются автоматически через sub-oval.online API
- Платежная система Platega (СБП, карты РФ, международные карты, криптовалюта)
- Реферальная система с начислением процентов на баланс
- Баланс пользователя для оплаты подписок
- Административные функции: рассылки, статистика подписок

### Технологический стек

- Python 3.12+
- aiogram 3.x (Telegram Bot API)
- SQLModel + PostgreSQL (база данных)
- Platega.io (платежная система)
- sub-oval.online (VPN subscription API)
- Docker + docker-compose

## 1. Административные команды (только для админа)

Для использования административных команд ваш Telegram ID должен быть добавлен в переменную `ADMIN_IDS` в файле `.env`.

### 1.1 VPN подписки — Автоматическая генерация

**Важно:** VPN подписки теперь генерируются автоматически через API `sub-oval.online`.

Больше не нужно вручную добавлять VPN ссылки через команды бота. При покупке подписки:
1. Backend обращается к sub-oval.online API
2. API создает encrypted subscription link
3. Link сохраняется в локальной БД (encrypted_subscriptions)
4. Пользователь получает link для подключения

**Конфигурация API:**

Добавьте в `.env`:
```
VPN_SUB_API_URL=https://sub-oval.online
VPN_SUB_ADMIN_USERNAME=admin
VPN_SUB_ADMIN_PASSWORD=your_password
```

**Параметры подписки по умолчанию:**
- `max_devices` = 3 (из `DEFAULT_MAX_DEVICES`)
- `tags` = ["main"] (VpnTag.MAIN)
- `profile_title` = BOT_NAME
- `support_url` = SUPPORT_LINK
- `profile_web_page_url` = BOT_LINK

## 4. Тарифы и подписки

### 4.1 Описание тарифов

| Тариф | Длительность | Цена | Лимит устройств | Trial |
|-------|-------------|------|----------------|-------|
| Trial | 3 дня | 0 ₽ (бесплатно) | 1 | Только для новых пользователей |
| Monthly | 30 дней | 199 ₽ (по умолчанию) | 1 | Нет |
| Quarterly | 90 дней | 499 ₽ (по умолчанию) | 1 | Нет |
| Yearly | 365 дней | 1999 ₽ (по умолчанию) | 5 | Нет |

**Примечание:** Цены настраиваются админом через команду `/prices`.

### 4.2 Trial подписка

**Особенности:**
- Доступна только новым пользователям (`is_new = True`)
- После активации пользователь помечается как не новый (`is_new = False`)
- Бесплатно, длительность 3 дня
- Лимит устройств: 1

### 4.3 Платные подписки

**Особенности:**
- Могут быть куплены многократно
- Новая подписка добавляется к текущей (если есть активная)
- Оплата с баланса или через Platega

## 5. Платежная система (Platega)

### 5.1 Методы оплаты

| Метод | Platega код | Описание |
|-------|-----------|----------|
| СБП QR-код | 1 | Быстрая оплата через СБП |
| Банковская карта РФ | 2 | Карты российских банков |
| Международная карта | 3 | Карты зарубежных банков |
| Криптовалюта | 4 | USDT, BTC, ETH и другие |
| ЕРИП | 5 | Оплата через ЕРИП (Беларусь) |

### 5.2 Процесс оплаты

**Шаги:**

1. **Выбор тарифа**
   - Пользователь нажимает "💳 Оплатить"
   - Выбирает тариф (monthly, quarterly, yearly)

2. **Выбор способа оплаты**
   - 💰 Баланс — оплата с баланса пользователя
   - 💳 СБП QR-код
   - 💳 Банковская карта РФ
   - 🌍 Международная карта

3. **Создание платежа**
   - Бот создает платеж через Platega API
   - Генерируется payment_url для оплаты

4. **Оплата на сайте Platega**
   - Пользователь переходит по payment_url
   - Выполняет оплату (QR-код, карта, крипта)

5. **Проверка статуса ("Я оплатил")**
   - Пользователь нажимает кнопку "✅ Я оплатил"
   - Бот проверяет статус через Platega API
   - Если `completed` → выдает VPN ссылку

6. **Автоматическая выдача**
   - Подписка создается в БД
   - Пользователь получает VPN ссылку
   - Реферер получает процент на баланс (если есть)

### 5.3 Webhook от Platega

Platega отправляет webhook на URL `PLATEGA_WEBHOOK_URL` при изменении статуса платежа. Бот автоматически:

- Обновляет статус платежа в БД
- Выдает VPN ссылку пользователю
- Начисляет бонус рефереру
- Отправляет уведомление пользователю

- При регистрации каждый пользователь получает уникальный реферальный код (8 символов)
- Код сохраняется в поле `referral_code` модели User

**Использование кода:**
- Новый пользователь использует код: `/start CODE12345`
- Пользователь помечается как реферал (`referred_by_id`)
- Реферер получает бонусы при платежах реферала

**Начисления:**
- **Первое пополнение реферала (≥120 ₽):**
  - Реферал получает: 5 ₽
  - Реферер получает: 10 ₽
- **Все пополнения реферала:**
  - Реферер получает процент (20% по умолчанию)
  - Процент настраивается через `REFERRAL_BONUS_PERCENT`

### 6.2 Пример начисления

**Сценарий:**
- Реферер: Alice (referral_code: ABC12345)
- Реферал: Bob (регистрация через `/start ABC12345`)

**Bob покупает подписку на 1000 ₽:**
- Alice получает: 1000 × 20% = 200 ₽ на баланс
- Alice получает уведомление:

```
🎉 Реферал оплатил подписку!

👤 Bob (@bob_username)
💰 Сумма: 1000 ₽
🎁 Вам начислено: 20% (200 ₽)

Баланс обновлен автоматически.
```

### 6.3 Настройка

**Переменные окружения:**
- `REFERRAL_BONUS_PERCENT` — процент от платежей рефералов (по умолчанию: 20)
- `REFERRAL_CODE_LENGTH` — длина реферального кода (по умолчанию: 8)

## 7. Troubleshooting (решение проблем)

### 7.1 Платеж не прошел / статус pending

**Проблема:**
Пользователь оплатил, но статус платежа `pending`.

**Решение:**
1. Проверьте webhook URL (`PLATEGA_WEBHOOK_URL`) должен быть publicly accessible
2. Проверьте `PLATEGA_WEBHOOK_SECRET` — должен совпадать с настройками Platega
3. Пользователь может нажать "Я оплатил" несколько раз для проверки
4. Проверьте логи бота на ошибки webhook
5. Проверьте dashboard Platega на статус платежа

**Команды для проверки:**
- `/check_<payment_id>` — проверка статуса платежа
- `/payments <telegram_id>` — история платежей пользователя

---

### 7.2 VPN ссылка не работает

**Проблема:**
Пользователь получил VPN ссылку, но она не работает.

**Решение:**
1. Проверьте API sub-oval.online (health check)
2. Проверьте `VPN_SUB_ADMIN_USERNAME` и `VPN_SUB_ADMIN_PASSWORD` в .env
3. Проверьте сервер VPN
4. Пользователь может получить новую ссылку в профиле (нажать на 🔗 monthly)
5. Убедитесь, что подписка активна (`/subscriptions`)
6. Проверьте таблицу `encrypted_subscriptions` в БД

---

### 7.3 Пользователь не получил подписку

**Проблема:**
Платеж завершен успешно, но подписка не создана.

**Решение:**
1. Проверьте статус платежа (`/payments <telegram_id>`)
2. Если статус `completed`, но нет подписки — проверьте логи бота
3. Проверьте таблицу `subscriptions` и `encrypted_subscriptions`
4. Проверьте доступность API sub-oval.online
5. Попробуйте создать подписку вручную через БД или повторить платеж

---

### 7.4 Ошибка при рассылке

**Проблема:**
Рассылка завершена с ошибками.

**Решение:**
1. Проверьте Telegram ID пользователей с ошибками
2. Пользователь мог заблокировать бота — ошибка "Forbidden: bot was blocked by the user"
3. Увеличьте задержку между сообщениями (sleep(0.1) в коде)
4. Проверьте длину сообщения (max 4096 символов для Telegram)
5. Разбейте сообщение на части если слишком длинное

---

### 7.5 Trial не активируется

**Проблема:**
Пользователь нажимает "✅ Начать" для trial, но получает ошибку.

**Решение:**
1. Убедитесь, что пользователь новый (`is_new = True`)
2. Проверьте доступность API sub-oval.online
3. Проверьте `VPN_SUB_ADMIN_PASSWORD` в .env
4. Пользователь не должен иметь активных подписок
5. Проверьте логи бота на ошибки при создании подписки

---

### 7.6 Реферальный код не работает

**Проблема:**
Пользователь использует `/start CODE12345`, но не становится рефералом.

**Решение:**
1. Убедитесь, что код существует в БД (проверьте `referral_code` пользователей)
2. Код должен быть активным (не удален/деактивирован)
3. Пользователь должен быть новым (не зарегистрирован ранее)
4. Проверьте логи бота на ошибки при регистрации
5. Убедитесь, что `referred_by_id` установлен в БД для нового пользователя

---

### 7.7 Platega webhook не приходит

**Проблема:**
Platega не отправляет webhook при изменении статуса платежа.

**Решение:**
1. Проверьте `PLATEGA_WEBHOOK_URL` — должен быть publicly accessible (не localhost)
2. Убедитесь, что webhook URL настроен в dashboard Platega
3. Проверьте `PLATEGA_WEBHOOK_SECRET` — должен совпадать
4. Проверьте логи сервера на HTTP 500 ошибки при обработке webhook
5. Проверьте HTTPS сертификат на webhook URL
6. Используйте `/check_<payment_id>` для ручной проверки статуса

## 8. Примеры сценариев использования

### 8.1 Сценарий 1: VPN подписка через API

**Задача:**
Пользователь покупает подписку, backend автоматически генерирует VPN link через API.

**Шаги:**
1. Пользователь покупает подписку
2. Backend вызывает API `https://sub-oval.online/api/v1/admin/subscriptions/encrypted`
3. API создает encrypted subscription link
4. Link сохраняется в `encrypted_subscriptions` таблицу
5. Пользователь получает link для подключения

**Параметры API:**
- `tags`: ["main"]
- `ttl_hours`: зависит от тарифа (trial: 72, monthly: 720, quarterly: 2160, yearly: 8760)
- `max_devices`: 3 (по умолчанию)
- `metadata`: из .env (BOT_NAME, SUPPORT_LINK, BOT_LINK)

**Результат:**
- VPN link генерируется автоматически
- Link уникальный для каждого пользователя
- Истекает автоматически по TTL

## 9. Конфигурация

### 9.1 Обязательные параметры

| Параметр | Описание | Пример |
|----------|----------|---------|
| `BOT_TOKEN` | Токен Telegram бота | `123456:ABC-DEF1234ghIkl` |
| `ADMIN_IDS` | Telegram ID админов (comma-separated) | `123456789,987654321` |
| `PLATEGA_MERCHANT_ID` | UUID merchant ID | `abc123-def456-...` |
| `PLATEGA_SECRET` | API ключ | `secret_key_123` |
| `PLATEGA_WEBHOOK_URL` | URL для webhook | `https://yourdomain.com/webhook` |

### 9.2 Настройка админов

**Как добавить админа:**
1. Узнайте свой Telegram ID (можно через @userinfobot в Telegram)
2. Добавьте ID в `.env`:
```
ADMIN_IDS=123456789,987654321
```
3. Перезапустите бот

**Проверка:**
- `/subscriptions` — если бот отвечает, вы админ
- Если бот отвечает "❌ У вас нет прав", ваш ID не в списке

### 9.3 Реферальная система

| Параметр | Описание | По умолчанию |
|----------|----------|-------------|
| `REFERRAL_BONUS_PERCENT` | Процент от платежей рефералов | 20 |
| `REFERRAL_CODE_LENGTH` | Длина реферального кода | 8 |

### 9.4 Поддержка

| Параметр | Описание | Пример |
|----------|----------|---------|

| `BOT_LINK` | Ссылка на бот | `https://t.me/your_bot` |

### 9.5 Platega

| Параметр | Описание |

| `PLATEGA_SECRET` | API ключ (header X-Secret) |
| `PLATEGA_WEBHOOK_URL` | URL для получения webhook |
| `PLATEGA_WEBHOOK_SECRET` | Секрет для подписи webhook |

### 9.6 VPN Subscription API (sub-oval.online)
